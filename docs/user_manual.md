# Amazon FBA报表工具用户手册

## 1. 系统概述

Amazon FBA报表工具是一款专为亚马逊卖家设计的数据分析与报表生成系统，帮助卖家实时监控销售表现、利润情况，优化库存管理，提升业务效率。

### 核心功能：
- ASIN利润报表分析
- 市场与店铺性能对比
- 每日销售数据统计
- 企业微信日报自动推送
- Excel报表导出
- 实时数据监控与预警

## 2. 安装与配置

### 2.1 系统要求
- Python 3.8+
- Flask 2.0+
- SQLAlchemy 1.4+
- Pandas
- APScheduler
- 数据库：SQLite或MySQL

### 2.2 安装步骤

#### 后端服务安装：
1. 克隆代码仓库：
```bash
git clone <repository_url>
cd amazon-report-tool/backend
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

3. 配置数据库：
   - 修改 `config.py` 文件中的数据库连接信息
   - 初始化数据库：
   ```bash
   python init_db.py
   ```

4. 启动服务：
```bash
python run.py
```

#### 前端部署：
1. 进入前端目录：
```bash
cd ../frontend
```

2. 配置API地址：
   - 修改 `config.js` 中的API基础URL

3. 部署静态文件到Web服务器（Nginx或Apache）

## 3. 使用指南

### 3.1 登录系统
- 访问系统URL
- 使用管理员创建的账号登录

### 3.2 店铺管理
- 在「店铺设置」中添加亚马逊店铺
- 配置OAuth授权
- 设置店铺显示名称和区域

### 3.3 报表功能

#### ASIN利润报表：
- 选择日期范围、店铺、市场等筛选条件
- 查看ASIN详细的销售、利润数据
- 按不同维度排序（利润、销售额、订单量等）
- 支持分页查看大量数据

#### 市场性能对比：
- 对比不同亚马逊市场的销售表现
- 查看各市场的ASIN数量、订单量、销售额和利润
- 分析各市场的平均利润率和客单价

#### 店铺性能对比：
- 对比多个店铺之间的业绩差异
- 监控各店铺的整体运营状况
- 识别表现最佳和需要改进的店铺

### 3.4 企业微信推送设置

1. 进入「系统设置」→「通知设置」
2. 配置企业微信机器人Webhook URL
3. 设置推送开启状态
4. 配置推送时间（小时和分钟）
5. 测试推送功能

### 3.5 数据导出

1. 在任意报表页面，点击「导出Excel」按钮
2. 系统会自动生成包含详细数据和汇总信息的Excel文件
3. 导出的文件包含三个工作表：数据明细、市场汇总、总体汇总

## 4. 常见问题

### 4.1 数据同步问题
- 问：为什么我的销售数据不是最新的？
- 答：系统默认每小时自动同步一次数据，您可以在「同步设置」中调整同步频率或手动触发同步。

### 4.2 企业微信推送失败
- 问：为什么企业微信没有收到日报？
- 答：请检查Webhook URL是否正确、网络连接是否正常，以及推送时间设置是否合理。系统日志中会记录详细的推送状态。

### 4.3 报表加载缓慢
- 问：加载大量ASIN数据时报表很慢怎么办？
- 答：建议缩小日期范围，或使用更多筛选条件来减少返回的数据量。系统已启用缓存机制来提升性能。

## 5. 数据安全

- 所有数据传输采用HTTPS加密
- 用户密码采用加密存储
- 定期备份数据库
- 遵循最小权限原则的访问控制
