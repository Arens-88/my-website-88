# FBA费用计算器 - 数字签名指南

## 为什么需要数字签名？

数字签名是解决Windows SmartScreen拦截问题的最佳方案。当用户首次运行未经签名的程序时，Windows会显示安全警告，提示程序可能不安全，这会影响用户体验和软件可信度。

通过为程序添加数字签名，您可以：
1. 消除Windows SmartScreen的安全警告
2. 提升软件的专业形象和可信度
3. 确保用户能够顺利安装和使用您的软件
4. 避免被杀毒软件误报为恶意软件

## 获取代码签名证书

要为程序添加数字签名，您需要先从权威的CA（证书颁发机构）购买代码签名证书。以下是一些知名的证书提供商：

1. **DigiCert** - 业界领先的证书提供商，提供高质量的代码签名证书
2. **Sectigo** (原Comodo) - 提供经济实惠的代码签名证书
3. **GoDaddy** - 知名的域名和证书服务提供商
4. **GlobalSign** - 提供全球认可的代码签名证书

### 购买注意事项

- 个人开发者可以购买个人代码签名证书，公司可以购买EV（扩展验证）代码签名证书
- EV证书具有更高的信任级别，可以更快地建立SmartScreen信誉
- 证书通常有效期为1-3年
- 价格范围通常在$100-$500不等，取决于证书类型和有效期

## 使用签名工具

我们提供了一个方便的批处理脚本，帮助您轻松为程序添加数字签名。

### 使用步骤

1. 下载并安装您的代码签名证书（通常为.pfx格式）
2. 确保您的系统已安装Windows SDK（包含signtool工具）
3. 双击运行`sign_app.bat`脚本
4. 按照提示输入以下信息：
   - 代码签名证书(.pfx)的完整路径
   - 证书密码
   - 要签名的程序路径（默认为dist\FBA费用计算器安装程序.exe）
5. 等待签名完成

### 手动签名命令

如果您希望手动执行签名，可以使用以下命令：

```
signtool sign /f 您的证书.pfx /p 证书密码 您的程序.exe
```

## 验证签名是否成功

签名完成后，您可以通过以下方法验证签名是否成功：

1. 右键点击已签名的程序文件
2. 选择"属性"
3. 切换到"数字签名"选项卡
4. 如果看到签名信息，则说明签名成功

## 注意事项

- 保护好您的代码签名证书和密码，不要泄露给他人
- 签名前确保程序已经过全面测试，没有安全漏洞
- 定期更新签名证书，避免证书过期导致签名失效
- 签名后的程序不要修改，否则签名会失效

## 常见问题解答

**Q: 为什么我找不到signtool.exe？**
A: 请安装Windows SDK。您可以从Microsoft官网下载Windows SDK，安装时选择"Windows SDK Signing Tools for Desktop Apps"组件。

**Q: 签名成功后，仍然被SmartScreen拦截？**
A: 新证书需要建立信誉。随着更多用户下载并运行您的程序，SmartScreen会逐渐建立信任。EV证书通常能更快地获得信任。

**Q: 我可以使用自签名证书吗？**
A: 不建议使用自签名证书。Windows不会信任自签名证书，仍然会显示安全警告。请使用权威CA颁发的证书。
